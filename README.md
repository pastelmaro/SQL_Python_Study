# 📦 쇼핑몰 주문 통합 및 재고 관리 자동화 시스템 (Mini ERP)

## 1. 프로젝트 개요
**"반복되는 주문 수집과 재고 관리 업무를 자동화하여, 휴먼 에러를 0%로 줄인다."**

이 프로젝트는 엑셀로 관리되던 쇼핑몰 주문 내역을 Python과 SQL을 활용해 데이터베이스화(DB)하고, **주문 수집-재고 차감-발주 알림**까지의 물류 프로세스를 자동화한 프로토타입입니다. '이카운트'나 '셀메이트' 같은 ERP 시스템의 작동 원리를 직접 구현하며 물류 흐름을 깊이 있게 이해하고자 제작했습니다.

## 2. 핵심 기능 및 해결 과제
단순한 엑셀 작업에서 발생할 수 있는 치명적인 실무 문제들을 기술적으로 해결했습니다.

### ✅ 1. 중복 주문 방지 (Double Shipping Prevention)
- **문제:** 엑셀 취합 과정에서 주문 건이 중복 입력되어 상품이 두 번 발송되는 사고 위험.
- **해결:** 주문번호(PK)를 기준으로 중복 여부를 자동 검사하여, 이미 처리된 주문은 시스템이 자동으로 **[SKIP]** 처리합니다.

### ✅ 2. 실시간 재고 유효성 검사 (Inventory Validation)
- **문제:** 재고가 없는데 주문을 접수하여 고객 클레임(품절 안내) 발생.
- **해결:** 주문 접수 즉시 DB의 현재 재고를 조회하고, **재고 부족 시 주문 승인을 자동으로 거절(Block)**하는 로직을 구현했습니다.

### ✅ 3. 안전 재고 알림 시스템 (Safety Stock Alert)
- **문제:** 재고가 떨어질 때까지 모르고 있다가 배송 지연 발생.
- **해결:** 출고 후 남은 수량이 설정된 '안전 재고' 미만일 경우, **"발주가 필요합니다"라는 경고 메시지를 출력**하여 선제적인 물류 대응이 가능합니다.

## 3. 사용 기술 (Tech Stack)
- **Language:** Python 3.9 (데이터 처리 및 자동화 로직 구현)
- **Database:** SQL (SQLite - 관계형 데이터베이스 설계 및 트랜잭션 처리)
- **Library:** Pandas (대용량 엑셀 데이터 전처리)

## 4. 기대 효과
- 수천 건의 주문 데이터를 **1초 이내에 처리**하여 업무 효율 극대화.
- 엑셀 수기 입력 시 발생하는 **오기입, 누락, 중복 사고 원천 차단**.
- 데이터 기반의 정확한 재고 관리로 **재고 불일치율 감소**.
